My project

42서울 교육이 코로나로 인해 온라인으로 진행 되었습니다. 그에 따른 부작용으로 밥 먹을 교육생을 찾는 것이 쉽지 않았습니다. 어떻게 하면 밥 먹을 교육생을 재밌고 쉽게 찾을 수 있을지 고민하다가 랜덤채팅을 떠올리게 되었습니다. 그래서 소개팅처럼 얼굴을 대면 할 수 있는 환경을 만들어 해결하고자 했습니다.

데모 제작과정


42서울 교육장 출입 API 서버에 잦은 장애가 발생했습니다. 그래서 서버 장애가 발생하더라도 출입이 가능해야 했습니다. 출입서버에서 받은 UserInfo를 저장하고 그 정보를 기반으로 자체 JWT를 발행하여 최초 인증, 재로그인시를 제외하고는 출입서버의 장애와 관련없이 입장 할 수 있도록 했습니다.

데모 제작과정


42서울에서 블라인드처럼 익명성이 보장 된 채팅 공간의 필요성이 대두 되었습니다. 그에 따라 앱 환경에서 가볍게 사용 할 수 있는 채팅 앱을 만들고자 했습니다. AOS, iOS 모두 개발 하는 것은 대안으로 React Native를 사용해 구현하게 되었습니다. Spring MVC STOMP를 사용하는 공식 가이드를 바탕으로 구현했습니다. 그런데 빈번한 요청이 생길 것으로 예상되어 비동기를 사용하면 좋을 것 같아 Spring Reactive, Rsocket 사용을 검토했습니다. 하지만 많은 시도와 검색을 했지만 Rsocket은 메시지 프로토콜을 직접 정의해야 돼서 현재 사용하긴 어렵다고 생각했습니다. 최종적으로 앱에서 작동 되도록 하기 위해 React Native의 Expo를 바탕으로 프론트엔드를 구현했습니다.
